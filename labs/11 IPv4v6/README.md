### Выполнение работы.
#### 1. Распланировать адресное пространство
Общий принцип планирования:
для IPv4:
10.10.1.Х/24 - подсеть для управляющей VLAN777, где Х - номер устройства
192.168.Х.Y/24 - подсеть для конечных устройств, где Х - номер VLAN равный номеру конечного VPC
Для связи между роутерами внутри одной AS или сайта используем подсети 172.16.X.Y/30, где Х зависит от AS (Х=254 (AS1001), Х=52 (AS520), Х=42 (AS2042))
Для связи между роутерами из разных AS или сайтов используем подсети 195.178.0.Х/30

для IPv6:
fc00:777/32 - подсеть для управляющей VLAN777, последний октет - номер устройства
